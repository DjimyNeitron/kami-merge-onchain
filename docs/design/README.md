# Kami Merge — Design Documentation

This directory contains canonical design decisions for Kami Merge implementation.

## Files

- **Handoff_v13.md** — Operational state, session history, implementation roadmap
- **Brief_v13.md** — Strategic design decisions across 6 sessions (NFT economics, metadata, onboarding, share, backend, grant)
- **NFT_Lore.md** — Single source of truth for 44 NFT descriptions (11 base lores + 4 tier flavors)

## Versioning

When design decisions change:
1. Update relevant `vN.md` file with amendment OR create `vN+1.md` superseding it
2. Commit with message format: `docs(design): {what changed}`
3. Reference in next handoff doc
